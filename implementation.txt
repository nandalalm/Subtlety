## Stage 12: AJAX List/Unlist — No Page Reload

### Goal
Every "Unlist / Relist / Block / Unblock" button fires a `fetch()` call and updates ONLY
the affected table row in-place — no `location.reload()`.

### Row-update pattern
After successful AJAX toggle:
1. Swap the button's classes and text
2. Update the status badge in the same row
3. Show a toast/swal success for 1.5s

### Pages that need AJAX toggle (currently reload):
| Page | Toggle target | Current | Goal |
|------|--------------|---------|------|
| `admin/usersList` | Block/Unblock user | `location.reload()` | Row DOM update |
| `admin/products` | List/Unlist product | `location.reload()` | Row DOM update |
| `admin/categories` | List/Unlist category | `location.reload()` | Row DOM update |
| `admin/reviews` | List/Unlist review | `location.reload()` | Row DOM update |
| `admin/coupons` | List/Unlist coupon | `location.reload()` | Row DOM update |
| `admin/offer` | Unlist/Relist offer | `location.reload()` | Row DOM update |

### Implementation per partial
- Add `data-id` and `data-status` attributes to each toggle button
- Add `data-row-id` to the `<tr>` so JS can find it
- Write a generic `toggleStatus(url, btn, statusKey)` helper function in a shared JS file
  or inline per partial
- The helper updates button text/class and badge text/class without reload

## Stage 14: AJAX Pagination

### Goal
All paginated pages load the next/prev page by replacing the table container HTML via
`fetch()` — no full-page navigation.

### How it works
1. Every controller that renders a paginated page gets a parallel JSON route:
   e.g. `GET /admin/orders?page=2` returns HTML page,
   `GET /admin/orders?page=2&_ajax=1` returns JSON `{ rows: [...], totalPages, currentPage }`
2. The pagination partial renders anchor tags with `data-page` attributes
3. A shared JS listener intercepts clicks, fires `fetch(?page=N&_ajax=1)`, replaces the
   `<div id="table-container">` innerHTML with the rendered rows, and updates the
   pagination partial HTML

### Alternative (simpler): server-side EJS fragment rendering
- Add an `?_ajax=1` branch in each controller that renders ONLY the table partial (not
  the full page layout) and returns it as HTML
- JS fetches this HTML fragment and swaps the container div

### Pages requiring AJAX pagination:
- `admin/orderList` — orders table
- `admin/usersList` — users table
- `admin/products` — products table
- `admin/categories` — categories table
- `admin/reviews` — reviews table
- `admin/coupons` — coupons table
- `admin/offer` — offers table
- `admin/salesReport` — sales report table
- `user/orders` — user orders table
- `user/userWallet` — wallet transactions table

### Controller changes needed (for each paginated page):
- Add `if (req.query._ajax) { res.render('admin/partials/xxxTable', data); return; }` branch
- This renders ONLY the table partial, which is returned as HTML string

240: ## Stage 17: Cart & Checkout Refinement — Integer-First Pricing ✅
241: 
242: ### Goal
243: Standardize all monetary calculations to use pure integers (truncated via `Math.floor`) at every step of the calculation, 
244: ensuring absolute consistency between backend data, frontend UI, and third-party payment gateways.
245: 
246: ### Core Logic Refinement
247: - **Source-Level Truncation**: In `utils/helper.js`, the `discountedPrice` of any product—after applying percentage or flat 
248:   offers—is truncated *immediately* via `Math.floor()`. 
249:   - *Example*: A 10% discount on ₹299 becomes exactly ₹269 (not 269.1). This prevents decimal drift when multiplying by quantity.
250: - **Mental Model Change (Subtotal vs Discount)**:
251:   - **Subtotal**: Now represents the sum of **Original Prices** for all items.
252:   - **Offer Discount**: Specifically tracks savings from Product/Category offers (Original - Truncated Discounted).
253:   - **Coupon Discount**: Specifically tracks additional savings from applied Coupon codes.
254: - **Atomic AJAX Sync**: The entire pricing summary (Subtotal, Offer Discount, Coupon Discount, Delivery Fee, and Total) 
255:   is refreshed in one atomic step whenever quantities change or coupons are toggled.

257: ### UI/UX Improvements

259: - **Checkout Sidebar**: Individual product rows show their **Base Price** so the subtotal above the footer remains intuitive.
260: - **Dynamic Coupon Controls**: Successfully applying a coupon hides the input field and "Apply" button, replacing them 
261:   with a "Remove Registered Coupon" button.
